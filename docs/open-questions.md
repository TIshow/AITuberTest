## 【最優先】技術的リスク

### Q1. Web Speech API はオフラインで動くか？

- **背景**: AITuberKit の音声入力は Chrome の Web Speech API を使用している
- **リスク**: Chrome の Web Speech API は Google サーバーへ音声を送るため、**オフライン環境では動作しない可能性がある**
- **検証方法**: Mac の Wi-Fi をオフにして AITuberKit の音声入力を試す
- **Fallback**: 動かない場合はローカル Whisper（Whisper.cpp 等）への差し替えが必要
- **決定期限**: Phase 0 の最初に確認する

### Q2. Ollama の日本語応答品質は十分か？

- **背景**: ローカル LLM は外部 API（Gemini 等）より日本語品質が劣ることがある
- **リスク**: 聴衆の前で見せるデモとして品質が不十分な場合、印象を損なう
- **検証方法**: Ollama で llama3.1:8b / qwen2.5 / gemma3 等を試し、教育テーマの質問に回答させて品質を評価する
- **Fallback**: 品質が不十分なら会場ネットワークを前提に Gemini 等の外部 API 利用に切り替える
- **決定期限**: Phase 0 で複数モデルを比較する

### Q3. RAG は必要か？システムプロンプトの直接埋め込みで足りるか？

- **背景**: 教育関連文書の量次第で対応方針が変わる
- **論点**: 文書が少量（数千トークン以内）なら RAG なしでも対応可能
- **検証方法**: 用意する教育文書のボリュームを先に確認し、プロンプトに収まるか試す
- **決定期限**: 教育文書が揃い次第

## アバター

### Q6. VRM アバターをどれにするか？

- **選択肢**: 既存の無料 VRM モデル / 有料モデル / 自作
- **考慮点**: 講演の雰囲気に合う見た目か。ライセンスは商用・講演利用 OK か
- **注意**: Live2D は Cubism SDK の商用ライセンス費用が発生する可能性あり → VRM を推奨

## 操作・本番運用

### Q8. 起動・準備の手順をどこまで自動化するか？

- VOICEVOX / Ollama / AITuberKit の3プロセスを毎回手動起動するのは手間
- 起動スクリプト1つで全部立ち上げる仕組みを作るか

### Q9. 誤発話・無関係な質問への対処はどうするか？

- 聴衆やマイクノイズが拾われて意図しない発話が起きる可能性
- 「AI時代の教育以外の質問には答えない」をプロンプトで制御するか、UIレベルで対応するか

### Q10. 応答が長すぎた場合の制御は？

- LLM が長い回答を生成すると、講演の流れを止めてしまう
- `max_tokens` で制限するか、プロンプトで「簡潔に答えて」と指示するか
